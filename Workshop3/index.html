Step 2 - Make a Variable

A variable is what we use to store things that we want to use in our code. For example, we can have a variable called price that stores how much money it costs to buy something in our store - lets say $50.

Type the following JavaScript into the console and press Enter. (You only need to type let price = 50) 


> let price = 50
< undefined

Now, 'price' holds the value 50. You can see this by typing price into the console. 


> price
< 50

Now let's give a name to this product that we can show as a title on our store - to use words as the value of the variable, we use quotation marks (e.g. 'words go here'), this type of value is called a string.

Try the following in the console.


> let title = 'Bread'
< undefined

Now we have a variable called title with the string value 'Bread'!

Bonus: How can you view the value of title?

Just like we did before with price, we just use the name of the variable to get it's value 


> title
< 'Bread'

That's great, but they're called variables because they can 'vary', that is, we can change the value of a variable by reassigning it.

Try it out by setting title to something else you want to sell. 


> title = 'Cake'
< 'Cake'

We don't need to use let here because we aren't making a new variable, we're using an existing one.

Bonus: Sometimes your variables can't vary!

There's actually another way of making a variable, instead of using the let keyword, we use const.

This makes a variable that we can't reassign.


> const dontVaryMe = 50
< 50



> dontVaryMe = 100
❌ Uncaught TypeError: Assignment to constant variable.

This might not seem particularly useful, but many programming bugs come from variables changing when we don't expect them to, so it's often nice to 'save us from ourselves' by making sure a variable doesn't change.

There's actually a third way of making a variable, using var, but we don't use that because it's evil! Bad things happen when you practice in the dark arts of var.

Step 3 - Use your variable!

Now let's make use of these variables by making a popup to show them.

Popups are shown using the alert() function. Functions are like variables, but instead of holding values, they hold more code! alert() has code to show a popup on our website, all we have to do is put the string we want it to show in the brackets ().

Try it out! Get alert() to say 'Hello World', remember that we use quotation marks '' because it's a string. 


> alert('Hello World')
< undefined
Bonus: What's this undefined thing?

Functions like alert() run code, but may also return a value. Just like how a variable gives it's value when used, a function gives it's return value.

When a function does not have a return value, like in alert()'s case, it's return value is undefined.

On the other hand, the function isFinite() returns whether or not the number you give it is finite, i.e., not infinity. 


> isFinite(10)
< true



> isFinite(Infinity)
< false

true and false are the return values (And just mean yes or no).

That's pretty cool! We can also add the values of our variables to the string that alert() shows.

Let's show an alert about how much our product costs. We use a plus symbol + to add our variables into our string, have a go at the following. 


> alert('The ' + title + ' is $' + price)
< undefined

It may look a little confusing, but all we're doing is adding the value of our variables into the string.

Step 2: Accessing Properties of Objects in an Array

We've learned how to log our array to the console, and how to access individual items of the array and log those as well. But what if we don't want to log an entire object, and instead only want to log the value of one property, such as "title"?

Individual properties of an object can be accessed using something called dot notation. We've seen this before in Level 1 step 4 of this runsheet, so scroll up and read through again if you need to refresh your memory.

We can combine the technique we just learned with dot notation by selecting the array item we want to access using its index, then typing a full stop followed by the name of the specific property you want to log after the square brackets.

In the example below, we are logging the title property of the first item in the products array (remember, the first product has an index of 0).

Now that we have learned how to access the properties of an individual object in an array, what if we want to change a property in some way?

Earlier in this runsheet (Level 1, Step 2), we talked about reassignment. We learned that even if a variable already has a value assigned, we can assign a new value and overwrite the previous one. The same thing will work with properties of objects.

Step 3: "For" Loops

So far, we have explored the concept of arrays, and have seen how to access a single item from an array using its index. We've also learned how to access a single property of an object using dot notation. If we want to access one property of ALL the items in the array and log all of them to the console, we can use something called a "for" loop to achieve this efficiently.

A loop is something that iterates (repeats over and over again) until a condition is met. A "for" loop is a specific kind of loop. Try removing any console logs you have remaining in your code, and add the code block below your products array: 


for(let i = 0; i < products.length; i++) {
    console.log(products[i].title)
}

If we break this down, the "for" loop in the example above essentially says:
Create a variable called i and set its value to 0. Then:

  1. Check if the value of i is less than the length of the array called products.
  2. If it is, access the title property of the object in the products array with an index equal to i and log it to the JavaScript console.
  3. Increment i by one.
  4. Repeat from step 1 until the value of i is no longer less than the length of the products array, at which point stop.
What exactly does the code inside the brackets mean?

There are three seperate expressions inside the brackets.

  1. let i = 0; This is an initialisation expression, which is creating a variable called i, and initializing it to 0. It executes before the loop starts, and is being used here to create a variable that can be used as a counter to keep track of how many times the loop has executed.
  2. i < products.length; This is a condition. It describes under what circumstances the loop should execute. If the condition is met, the loop will execute, otherwise it will stop. In our example, it means that if the value of i is less than the length of the products array (or in other words, less than the number of objects in the aray), the loop should execute.
  3. i++ This is a final-expression, which executes at the end of each loop iteration. Normally this is used to add 1 to the counter. In our example, it simply means "increment i by 1".

When you reload the page and check the console, the output should look something like this:

Step 4: Functions

A function is a block of code designed to perform a task. It will not be executed automatically; it needs to be invoked.

Functions are useful because they allow us to separate our code into segments representing different tasks, and reuse these segments in different situations. Let's try turning our "for" loop into a function. To do this, we need:

  • The "function" keyword
  • A name for our function.
  • Normal brackets containing parameters (a parameter represents anything our function needs as input- in this case, it needs an array of products)
  • Curly brackets to hold the actual code that makes up our function (in this case, our "for" loop)

First, add the following to your script.js file:


function addProductCard(products) {

}

Next, cut and paste your "for" loop so that it is contained within the curly brackets of your function, as shown below: 


function addProductCard(products) {
    for(let i = 0; i < products.length; i++) {
        console.log(products[i].title)
    }
}

There's only one thing left to do, which is to invoke your function by typing the function name (addProductCard) underneath, followed by brackets. Make sure you pass the products array to the function by including it in the brackets. The completed code is shown below: 


function addProductCard(products) {
    for(let i = 0; i < products.length; i++) {
        console.log(products[i].title)
    }
}

addProductCard(products);
Function and Parameter Names

Functions can be named whatever you want. It's best practice to create a name that describes what your function does, and they should always be written in camelCase (lower case first letter, and after that use an upper case letter to start each new word). Something like "logProductTitles" would be perfect here, but we have used addProductCard instead because we're going to be changing what this function does a little later on.

Parameter names should be all lower case, and should describe the thing you are passing to the function. In this case, we are passing an array of products to our function.

      const title = document.createElement("h2")
      title.innerText = products[i].title
      card.append(title)

      // Add price
      const price = document.createElement("p")
      price.innerText = "$" + products[i].price
      card.append(price)

      // Add the cards
      parentElement.append(card)
  }
}

You finished the final level! That's it for the base content, but if you're looking for more, try out the Extensions! 🗿

Extensions

Horizontal Scroll

Right now, if we have more than 5 products, the list gets pretty squished in the page. One solution to this is allowing our list to 'overflow' off the page. Once you have the products overflowing, the page automatically adds in a scroll bar to allow you to see everything.

We'll be adding a new class called 'scrollable' which will contain all the CSS going forward. First lets make sure our Top Products list belongs to this new 'scrollable' class by adding in the highlighted word in the HTML like so: 


<section>
    <h2>Top Products</h2>
    <div id="topproductlist" class="itemlist scrollable"></div>
<section>

Now we'll move onto our CSS and define this new class. We'll add in overflow-x which allows for horizontal overflow on our page like so: 


.scrollable {
    overflow-x: auto;
    justify-content: space-between;
}

However, the horizontal overflow won't work correctly unless we state that an itemcard needs to have a minimum width. This is because flexbox will by default squish the cards to fit the space provided. To fix it we'll add another bit of css code as below: 


.scrollable .itemcard{
    min-width: 300px;
}

To properly test this, you may need to add more products to your current products[] list in your Javascript code. Try adding in a bunch more products now that you don't have a limit.

Scroll Styling

The default scroll bar is a little bland, so to spruce that up we can use some more CSS that targets specifically the scroll bar within our product list.

There are 3 things we'll be changing for the scrollbar. The scrollbar itself, the the track of the scrollbar, and the draggable scroller (or thumb) that's on the track. We'll be adding the three bits of CSS to address these parts with some settings that suite our default style. Feel free to tweak the values to make it suit your own unique theme.

Add in the CSS below, save your files, and refresh your page to see the difference: 


section ::-webkit-scrollbar {
    width: 20px;
}

section ::-webkit-scrollbar-track {
    -webkit-box-shadow: inset 0 0 6px #c0c0c0;
    border-radius: 10px;
}
section ::-webkit-scrollbar-thumb {
    -webkit-box-shadow: inset 0 0 6px #c0c0c0;
    border-radius: 10px;
}

You may have noticed that ::-webkit-scrollbar isn't like any CSS code we've used before. :: denotes a 'pseudo-element', and web-kit is used for rendering content within Safari and Chrome browsers.

Auto-Scrolling

Now we're going to get into something a bit more complicated. We're going to animate our products list to scroll on its own, and when it hits the end of the scroll, we'll make it come back to the start.

To start, we'll add in a couple variables in our Javascript file. We'll add in a variable for the amount scrolled creatively called scrollAmount, and a variable for whether or not we are scrolling called isScrolling which we will have set as true, so that automatic scrolling is on by default. 


let scrollAmount = 0;
let isScrolling = true;

This code can be put at the bottom of your file by itself, it shouldn't be inside a function.

Underneath these 2 statements we'll have a function for the animation of the automatic scrolling itself. The code can be found below: 


setInterval(function () {
    const pauseMargin = 100;
    const total = topProductsElement.scrollWidth - topProductsElement.clientWidth + pauseMargin;
    if (isScrolling) {
        scrollAmount = (scrollAmount + 1) % total;
        topProductsElement.scrollLeft = scrollAmount;
    } else {
        scrollAmount = topProductsElement.scrollLeft;
    }
}, 10);

The logic here might be hard to understand, and that's okay, this is the Extensions section. If you're curious, it is essentially adding 1 to the scrollAmount to keep track of how far it has scrolled across, and uses topProductsElement.scrollLeft to go to that spot. It uses modulo, or % in code form, to tell if it has reached the end, and loops back to the start if it has. The pauseMargin is how long it waits before snapping you back to the beginning.

Assuming your code works, the products list should automatically scroll to the right, but what if you're trying to click on a product? you'd want it to stop right? To do that we can add in some more Javascript code for what the page should do when your mouse is hovering over the product list. The code is as below: 


topProductsElement.onmouseover = function () {
    isScrolling = false;
};

topProductsElement.onmouseout = function () {
    isScrolling = true;
};

Make sure this code is placed underneath your statements for isScrolling for this to work. What this code does, is it sets isScrolling to false when the mouse is hovering over our list, and once it isn't hovering over it anymore, it goes back to being set to true. This will stop and start the animation as desired, otherwise the mouse would stop it and the animation would stay off.
