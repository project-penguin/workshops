Level 1 - Semantic elements

In HTML we have something called semantic elements. These elements indicate the type of content that they contain. Which means that their content is defined clearly to both the browser and the developer (you!).

For instance, semantic elements like <table> and <section> describe the purpose of the element and the type of content that is inside them. On the other hand, non-semantic elements like <div> and <span>don't tell us anything about its content.

Why do we need it?

Firstly, it is easier to read and understand the code. Secondly, it's creating a better experience for users as search engines and assistive technologies (like text to speech for users with reading disabilities) are able to better understand the content of your website.

Usage

The <section> element defines a section in a document and can be used for: chapters, introductions, news items, contact information. The <div> tag defines a division in a document and allows you to separately style a certain portion of your webpage (using classes or IDs, which will be covered later). When used as a container for HTML elements, it has no effect on the layout until it is styled with CSS.

The <section> </section> element just like the <div> </div> tag come in pairs. The content is written between the opening and closing tags.

Level 2 - Add CSS classes

In the last workshop, we learnt how to style a particular element like <p>. One problem is that the CSS rule would end up affecting every <p> element in the page. What if we wanted to style just a specific group of elements? CSS classes are a way to do this! A CSS class is specified through the class attribute in an HTML tag. For example: 

 
<p class="redParagraph"> </p>

To add CSS rules to this class, we would have to add a dot before the name of the class in the CSS file. For example: 


.redParagraph {
    color: red;
}

This CSS will change the text colour of the paragraph to red. Remember, only HTML elements with the class "redParagraph" will be affected by this CSS. In this next example, only the last 2 <p> elements will be red coloured.

Code


<p> Hi there. </p>
<p class="redParagraph"> Welcome to my shop! </p>
<p class="redParagraph"> We have many items on sale! </p>

Did you know that different types of elements can belong to the same class? For example

You may notice that you start getting a horizontal scrollbar you make your browser window smaller. Add overflow: hidden to make the cards shrink instead of adding a scrollbar. 


.itemcard {

Now if your screen is large (or if you zoom out in the browser), you will notice that the image doesn't fully fill the card. }

Level 4 - Flexbox layout

Flexbox (or just "flex" for short) is a type of layout that we can use to change the size and arrangement of different elements on our page based on the size of the container they are in. You can imagine a container as an invisible box that holds a number of items. This box has special properties- when you make it bigger or smaller, the items inside it also grow or shrink, and move around to fill the available space. This "flexible" behaviour is the reason why this layout is called "flex".

We can turn any HTML element into a flex container by adding a CSS class to it, and then setting the display property on that class to "flex".

Let's try it out by turning our section element into a flex container. Earlier in Level 2, we added the class itemlist to the section element in our HTML file, so that part is already done. Next, we need to add the following code to our style.css file:

        
.itemlist {
    display: flex;
    justify-content: space-around;
}

When you save your changes and reload the page, you should notice that the layout has changed a lot. Previously, the item cards within the section element were displayed from top to bottom, but now they are displayed from left to right. This is because the default direction for flexbox layout is row (left to right). It's also possible to change the direction to column (top to bottom), but in this case we'll be sticking to row.

In addition, the "flexible" qualities of the flexbox layout can now be seen. If you click and drag to change the size of your browser window, the size of the itemlist container will change and the item cards inside it will also change size to fill the available space.

Finally, we have added one more property: justify-content. This controls how the cards are arranged along the main flex direction (remember, in this case it is row or left to right). The space-around option ensures that each card has an equal amount of space around it, and as a result the cards are distributed evenly along the row.

There are a lot of different options you can use to change the way items are displayed in a flexbox layout. For a more detailed overview, check out the links in the Additional Resources section below!

Add styling for a featured item

CSS classes are great, but what if we wanted to style just one and only one element? For example if we wanted to add a border to one of the itemcards, using a class is fine but not necessary. A better way would be to use an id attribute. An id must be unique so it can identify one element on the page.

Let's add an id attribute to the first itemcard on the page. 


<div class="itemcard" id="specialCard">
    ...
</div>

To style an element with an id, use the hash symbol before its name in the CSS file. For example:

Code


#specialCard {
    border-width: 5px;
    border-color: yellow;
    border-style: solid;
    font-style: oblique;
}

Result

These 4 CSS properties change the font style and create a yellow solid border with a 5px width. Feel free to change the values or try some other properties!
