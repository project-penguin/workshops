Workshop 1 - Introduction to HTML & CSS

Goal: learn about HTML & CSS and create your first web page!

Introduction

HTML....? CSS....? Whaaaat?

HTML stands for Hyper Text Markup Language, and it's the standard programming language used to create web pages. CSS stands for Cascading Style Sheet, and it describes how HTML elements should be displayed on a page.

With HTML you define the content to show on the page, and with CSS you style that content to make it look good.

Why should you care?

Every website on the Internet is made using HTML and styled using CSS. Even this runsheet was made using HTML!

Level 0 - Understanding Web Pages

Basic Web Page Structure

Every web page is made of elements that contain information.

An element consists of:

  • An opening tag, e.g. <title>. This specifies what type of element it is.
  • The content, which can be either plain text, or other elements (this is called nesting).
  • A closing tag, e.g. </title>. This is the same as the opening tag except with an added '/'.

A tag looks like a word with angle brackets around it, for example: <title>.

The example below shows the basic skeleton of a HTML page. As you can see, it is made up of many nested HTML elements. Read the comments (the grey text) to see the purpose of each element.


<!DOCTYPE html>
<html>
    <head>
        <!--
            The `head` tag is used for storing "invisible to the
            user/visitor" content. This content may include the 
            page title, references to CSS and Javascript files, 
            and tags which Search Engines use to identify your 
            page and its contents.
        -->
        <title>Workshop 1 - Learning HTML</title>
    </head>
    <body>
        <!--
            The `body` tag is used for the content that is, in most 
            cases, visible to users. This content may include text,
            links, images, tables and much much more. Try it out!
        -->
        <footer>
            <!--
                The `footer` tag is rarely used in HTML. The tag is 
                usually located at the very bottom of the `body`, 
                and includes useful links and copyright information.
            -->
        </footer>
    </body>
</html>

You may notice an odd looking tag: <!-- -->. This is a comment block. Anything within these two tags will not do anything.

Building Your Website

Level 2 - Head (and Body)

For these exercises you'll be editing the index.html you created.

We'll provide example code for each step, but feel free to change things to whatever you want. It's your website after all.

Head and Body

As we saw previously, a HTML document has both a head and a body, just like you (hopefully)! Let's start by adding tags for the head and body to create our basic skeleton. Your file should look like this: 


<!DOCTYPE html>
<html>
    <head>
    </head>

    <body>
    </body>
</html>

Indentation is very important for readability of your code! Any element that is inside another element (called a child element) should be indented once in relation to its parent element. You do this by pressing the `tab` key on your keyboard when starting a new line.

Stylesheet link

Remember the CSS file you created before? We need to tell our HTML page where to find it.

In between the <head> tags, add a new line and add a <link> tag as shown below.


...
    <head>
        <link rel="stylesheet" href="style.css">
    </head>
...

Note that the tag has two attributes: 'rel' defines what type of document is being linked, and 'href' gives the file location.

Note that the link element doesn't have a closing tag. This is because it doesn't need any content. This type of element is called an empty element.

Title

The title of a webpage is what is shown in the browser window or tab at the top of your screen. It's not shown on the page itself, which is why it goes in the <head> instead of <body>. What is the title of the webpage you are on right now?

Level 3 - The Nav

Now we want to move onto the <body>, where we can make our navigation bar. We can do this using the <nav> tag.

When we open our <nav>, we want to put our shop's name inside. For this example, we'll go with 'ShopInc', but you can call this website whatever you want.

Code


<body>
    <nav>
        <h1>ShopInc</h1>
    </nav>

Result

Above you can see we are using <h1>, which is 'Heading 1'. Headings can go from <h1> up to <h6>. The higher the number is, the smaller the heading is.

Right now it looks pretty basic. In fact, it's hard to tell there's a nav there at all. We can use CSS styling to make it look a whole lot better.

Go into your style.css file in VS code, and make an entry for your nav like so: 


nav {

}

This is called a selector. This means whatever code we put within the curly braces here will be applied specifically to our nav.

To improve our nav, let's add a background colour for it, and also add some padding so that there's space between the nav and elements underneath it in future. 


nav {
    background-color: #666666;
    padding: 20px;
}

In this example we've set background-color to #666666. Colours can be specified a few different ways, what we used here is known as a 'hex colour code', which are values between #000000 (black) and #ffffff (white).

Hue/Saturation/Lightness is another way to get specific colours, and can be performed by having hsl(0, 0%, 0%) up to hsl(360, 100%, 100%) in your CSS code instead. You can also use RGB (Red/Green/Blue) values with rgb(0,0,0) up to rgb(255,255,255), or in some cases you can just type in the colour with just words e.g. red, maroon, and darkslateblue.

Which technique you use is up to personal preference. If you want to pick your own colours for your website, you can get all of these values for a huge variety of colours here: W3Schools Colour Picker.

But let's get back to our code. We aren't done with CSS for our nav just yet, we also want to have some styling for the h1 that is inside the nav like this: 


h1 {
    color:  #93f6ff;
    margin: 0;
}

You can see we've used another hex value as our colour, feel free to change these to whatever you want. The margin: 0 line is to remove the default spacing that sorrounds the h1 in your HTML code by setting it to 0. This makes the nav look a bit sleeker overall.

The last thing we want to do, so that the nav covers the whole top of the screen (with no white space around it), is to set the margin in the <body> to 0 like this: 


body {
    margin: 0;
}

Make sure you save your style.css code and refresh your shop page in the browser to see the changes. If you applied these 3 selectors successfully in your CSS code (body, h1, nav) your page should look something like this:

You might prefer to have an image that serves as your shop logo instead of a heading. We're going to get into images in the next level, so feel free to come back and edit the contents of your nav later.

Level 4 - Did someone say headers?

What is a header?

Headers usually represent a container for introductory pictures or texts, or even a set of links to help the users navigate your cool website. You can think of it as a place to put introductory stuff to your website. Let's add a header, right underneath the nav that we created before.

Code


<header>
  
</header>

Now that we have a header section within our code, let's give a shot at adding a picture that your users will first see when they load up your website. Within your header section we will need to add an image link to add in a picture. We can do so by adding an img block within the header section.

Code


<header>
    <img src="images/shop-hero-banner.jpg" width="100%" alt="Shop">
</header>

Before we do so, let's get organised together and make a folder specifically to contain images. Navigate to your working folder within your file explorer and make a new folder called images. That way when we have more images in the future, we know where we can put them and where we can find them if we need them to display on our website.

We want to have a large image to use as a 'hero banner' for our shop now. See if you can find a cool image on the internet that suits your chosen style!

Alternatively, you could just right click and save the image below which we're using as an example, and change it later when you feel like it.

No matter what image you pick, make sure you save it in your new images folder so that future steps work for you

Now that you have a folder called images, and an image file to use sitting within it, you may be wondering what do src, width and alt mean within the <img> block?

How to set the attributes in the img block

src stands for source, which tells your website where to look for your image. In order to set the image for the img block, let's try to reference an image that we've put into our images folder. To reference your image, you need to specify its path. Start typing the folder name, i.e. images, and see if Visual Studio starts suggesting files through its auto-fill.

width stands for width, which tells your website how wide your image should be. From the example above, 100% sets the image to 100% of the original image. If you would like the image to be smaller, you could set it to 50% of the original image, which translates to half the original size! Another important way you could try setting your image width is by using px unit which stands for pixel. Pixels are relative to each device, which means one pixel on one device could be a different size on another device. To read more on CSS units, you can do so over here.

alt stands for alternate text, which specifies the text of an image if the image cannot be displayed. The alt text adds a text description to an image on a web page, and should be used wherever possible to ensure accessability. For people with vision impairments, alt text is accessed by their screen reader to provide them with a text equivalent of images.

You might notice that your banner image is taking too much room on your page. A quick and neat way to fix this would be to add in some CSS that targets images that are in your header like so:


header img {
    max-height: 500px;
    object-fit: cover;
}

The max-height statement controls...you guessed it, the maximum height of the image. Its set to 500px but feel free to tweak it to what works for you.

The object-fit is a quick statement to say you want the image to cover the full page, so horizontally it reaches the edges. Depending on the image you use, it might look weird because its favouring the center of your image Why not try out some of the other object-fit types like fill or contain and see how it changes.

Level 5 - Let's get some info up in this space!

Now that we've got our "Hero Banner" and our shop logo looking amazing, it's time to add some more content to our website. We'll start by adding a section of text where we can tell everyone who visits just what our store is all about!

Paragraphs

In the last level, we put the header tag into the <body> section of your HTML. We will be placing the next bit of code underneath this in the same section.

Let's start by adding the tags for a paragraph <p></p> which will hold our information.

Inside these tags type out what you'd like to say about your shop. For example: 


<p>Welcome to our shoppe!</p>

If you'd like to separate your information out and have multiple paragraphs, then it's easy. Just put each paragraph inside a new set of paragraph <p></p> tags!

Save your code and refresh your browser to watch the information you've written appear on your website!

At the moment, our text looks a little boring, and that's because we're using the default styling of your web browser. Well, thanks to the magic of our CSS stylesheet, we can make our text more interesting than a grey block of text!

Since we want our styling changes to apply to all the paragraphs on our page, it's important that we make our changes in the right section of our CSS. Working in our style.css file, we want to create a p section with curly braces, the same way we did in level 4.

Let's start by deciding how close to the left of the page we'd like our text to sit. We do this by adding a margin and giving it a value. 


p {
    margin: 20px;
}

Try experimenting with different numbers after the margin until you find a position that you're happy with.

Don't forget to save your work and refresh your browser each time so your changes appear.

Now that we've worked out the position of our text, how about changing the font?

We can specify the font we'd like to use or allow our browser to select the closest font to it using font-families in CSS. There are five generic font families which contain a list of different fonts: Serif fonts (which have a small stroke at the edges of each letter), Sans-serif fonts (which have clean lines), Monospace fonts (all letters have the same width), Cursive fonts (which look like handwriting), and Fantasy fonts (which are the decorative ones).

We want to apply our font to the whole body, so that it applies to all text including headings, not just our paragraphs. Going back to where we styled our body, add a line for font-family. Then we specify the font we'd like, followed by the font-family it belongs to. For example 'Trebuchet MS', sans-serif. 


body {
    margin: 0;
    font-family: 'Trebuchet MS', sans-serif;
}

Have a play around with the different font families and fonts available to you until you find one you like the look of. We go into this in more detail in the Extensions below.

If you have time, and you want to try out something extra (like changing font size or colour) move on to the Extensions section below.

If you don't want to use any of the standard HTML fonts and you're looking for something more specific or interesting, there is another solution - Google Fonts .

The thing to bear in mind with these fonts is you have to include a reference to Google's own font information for it to work.
For example, let's say you wanted to use the 'Comic Neue' font from google fonts, then you'd need to add a reference like this into the head of your HTML code (the same way you added a reference to you CSS file earlier): 


<head>
    ...
    <link rel="stylesheet" href="https://fonts.googleapis.com/css?family=Comic+Neue">
</head>

And just as you would with a standard HTML font, in the body{} section, replace the font-family value with the name of your chosen font. So for this Google Fonts example it would be: 


body {
    ...
    font-family: 'Comic Neue', cursive;
}

Styling Specific Text

We applied a font-family to the body{} section of your HTML, which means it applies to all text that displays on your page. If you want to specify a font-family for only Paragraph text for example, you would need to put the 'font-family:' code line into the p{} section instead.

The font-weight property dictates how thick the individual text typed on your webpage will be. Its value can be set with numbers between 100 and 900, or with words like normal, bold, bolder, lighter.
400 is the same as normal, and 700 is the same as bold. As an example, let's change our paragraph text to have a weight between normal and bold by adding the line font-weight: 550 in the p{} section of our code.
Another thing to note is that a given Google Font may only have styles for certain weight numbers. For example, Comic Neue only has styles for weights 300, 400, and 700.

The font-size property determines how large you want the text to be on the page. It can take values from 'xx-small' to 'xx-large', or any other css units like '10px', '50%' or '3em'.
Let's set our paragraph's text size to be bigger than the default using px (pixel) units. We just need to add the line font-size: 20px in the p{} section of the code. Change it to whatever looks good to you, just bear in mind that 16px is the default font size.
